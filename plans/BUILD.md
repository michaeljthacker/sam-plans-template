# BUILD — Template

## Purpose
`BUILD.md` describes the **multi-milestone plan for the current Build**:
- what is being built and why,
- high-level scope,
- milestones and success criteria,
- notable constraints and assumptions.

A Build is a major body of work on the product.

## When to update
- When starting a new Build
- When Build-level scope changes
- When milestones are added/removed/re-ordered

## Required sections
- Purpose
- Scope (in / out)
- Success criteria
- Milestones (list)
- Risks / assumptions (optional but recommended)
