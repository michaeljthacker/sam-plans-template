## About this file
`BUILD.md` describes the **multi-milestone plan for the current Build**:
- what is being built and why,
- high-level scope,
- milestones and success criteria,
- notable constraints and assumptions.

A Build is a major body of work on the product.

## When this file is updated
- Created by `Product.ProductVision` at the start of a new Build
- Reviewed (not modified) by `Principal.BuildReview`
- Updated when Build-level scope changes or milestones are added/removed/re-ordered

## Expected sections (generated by the action)
- **Build ID** — e.g., B1
- **Purpose** — what is being built and why
- **Scope** — in-scope and out-of-scope items
- **Success criteria** — measurable outcomes
- **Milestones** — ordered list (M1, M2, …)
- **Risks / assumptions** — known risks and assumptions (recommended)
